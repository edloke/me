L'Histoire

2024 • UI Design Project

L'Histoire is an eBook app that allows readers to be able explore and read through a wide collection of different books and novels. This app aims to provide a one stop solution for readers to be able to store their existing library as well in the app as well as adding new books to read into their collections through the app.

Colour Palette

Here are the colour palette that defines the overall branding and presence of the user interface which is being used throughout the design.

Typography

Inria Serif is used throughout the user interface design. During my thought process, I came to the conclusion of using this font as I felt that this font could best express what the application really is and provides the users an experience that is similar to reading a physical book.

My Library Page

This page has been designed with several considerations in mind which is the main focus and essentials that could help users to continue where they left off from their previous reading.

  1. This section is created to allow users to have quick access to their current readings and allow them to continue where they left off easily.

  2. Access to the whole library, read list, history and bookmarks are placed in this section to allow users to easily browse through their collection. Graphical elements are also used to assist and improve accessibility.

  3. The search button is placed on the top of the page to allow users to easily search through their library.

My Account Page

During the process of designing this page, I have made several consideration on where to include the goals functionality that could help the user best keep track with their progress.

  1. A goal summary is added to the account page to allow users to have quick view of their current goal progress which helps them to keep track of their goals.

  2. The user's profile name and picture is displayed on the top of the page along with a customisable backdrop that helps user to personalise the application to their own preference.

Refinements

My Library - Changes

In the My Library page, there are several changes that were made during the transition of Lo-Fi prototype to Hi-Fi prototype.



The main changes would be the search bar which was previously just a search button. The reason for this change is to provide a better cognition ability for the user where it better represents to the user that they could browse their library through the search bar.



Another change made was towards the book tile where the chapter where the user left off is listed as compared to the initial mockup that only indicates that it is in progress.



In addition, the progress bar is also moved to the bottom of the tile to provides a less distracting experience when going through previously read books.

Goal Progress Page - Changes

In the Goal Progress page, there are several changes that were made during the transition of Lo-Fi prototype to Hi-Fi prototype.



One of the changes would be introduction of a card style to display the goal and added the flexibility to view all previous goals from that page.



Another change is the introduction to a backdrop behind the card as well as a greeting message that provides a more welcoming and fun experience to the user.



Furthermore, there is a new user's achievements tab added on the top right along with their profile picture which is absent from the initial design. This achievement tab allows users to view their current achievements and look at the requirements to earn a new achievement.

Accessibility Considerations



Text Size Adjustment

When designing the interface, I have taken into consideration of the primary task that users will use the app for which would be reading.

Hence, in order to improve accessibility I have included the function to adjust the applications text size into my design. The main purpose of this design consideration is to allow users to adjust the text size for their reading based on their preference.



Heading Hierarchy

Another accessibility consideration that has been included in this design would be the heading hierarchy where the placement of the each text has been deeply taken into consideration when designing the interface. For example, having a larger and bolded text before a text that is smaller.



Going Forward


What I have Learned?

From this project, I found that there are many things to consider when designing a user interface such as the number of features and the userflow of each feature. Hence, there must be planning that have to be done before starting designing such as what are the different user flows that would exist in the application and how it should be implement and if it could provide a better user experience to the user.


Next Steps

Continue to explore new designs and keeping up with the latest trend. In addition, going forward I will try to take on more challenges in creating different user interfaces.


Doing a more in-depth research on accessibility and learn more about WCAG standards and what elements that could be used to help improve the overall user experience.
